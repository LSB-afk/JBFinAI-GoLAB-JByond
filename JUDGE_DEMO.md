# 심사위원 시연 가이드 (5분)

JB LocalGuard OS — 담당자 승인 중심 금융 AI 에이전트 운영 콘솔. 프레임워크·빌드 없이
브라우저에서 전체 운영 루프(접수 → 분류 → 에이전트 제안 → 사람 승인 → 감사 기록)가 동작합니다.

## ① 준비 (1분)

- 요구사항: `python3`, `node` (버전 무관, 정적 서버 + Playwright만 사용)
- 설치:
  ```bash
  npm install          # Playwright (검증용, 시연 자체엔 불필요)
  ```

## ② 구동 (택1, 셋 다 무료·무키)

```bash
# 기본 — 정적 프로토타입만으로 전체 데모 완주
npm run dev          # → http://127.0.0.1:8000/index.html

# 백엔드 데모 — /api/* 포함 (역할·케이스·감사 로그 API)
npm run backend      # → http://127.0.0.1:8010/index.html

# 라이브 옵션 (선택, 없어도 fallback으로 완주)
npm run demo:proxy   # 실거래가 API 프록시 :8020 — ?live=1 용
npm run demo:ollama  # 로컬 모델 프록시 :8030
```

## ③ 핵심 시연 동선 (3개, 각 1분)

**1. RM 역할 하네스 콘솔** — 역할 레일에서 담당자 역할 선택 → 보드 진입.
보여줄 것: 역할별로 완전히 분리된 업무 하네스(계열사·역할이 곧 독립 콘솔). 말할 포인트:
"역할마다 별도 데이터 스코프 — 기업여신 담당자는 보이스피싱 케이스에 접근 자체가 차단됩니다."

**2. 기업여신 심사지원 포털** — `#/roles/corporate-credit/board`
보여줄 것: 재무 요약·상환 체크·서류 확인·품의 초안까지 AI가 제안하고, 승인/거절·금리/한도는
버튼 자체가 담당자 전용. 말할 포인트: "AI는 초안까지, 최종 승인 버튼은 사람만 누릅니다."

**3. 전세사기 보호 업무지원 포털** — `#/roles/jeonse-protection/board` (`?live=1` 추가 시 실거래가 반영)
보여줄 것: lifecycle 보드 · 공공데이터 시세 엔진 · 위험 신호 스코어링 · 피해지원 연계.
말할 포인트: "국토부 실거래가 API를 직접 붙였습니다 — 키가 있으면 실데이터, 없으면 동일한
시뮬레이션 값으로 자동 대체됩니다."

(여유 있으면 `?demo=sme`/`?demo=phishing` 홈 CTA로 GP-1/GP-2 Hero 시나리오, 또는
`#/jb-woori-capital/board`로 계열사 확장성도 보여줄 수 있습니다.)

## ④ 폴백 규칙

- 키·로컬모델이 전혀 없어도 **결정론적 오프라인 시뮬레이션**으로 완주합니다.
- `?live=1` — 실거래가 프록시(`demo:proxy`)가 떠 있으면 실데이터, 없으면 자동 시뮬레이션 fallback.
- `?model=0` — 로컬모델 플래그만 끄기 (`window.RUNTIME_CONFIG`).
- 감사 로그에 시세 출처(`공공데이터` vs `시뮬레이션 입력`)가 그대로 기록되어 무엇이 실동작인지 투명합니다.

## ⑤ 트러블슈팅

- **포트 충돌(8000/8010/8020/8030)**: 기존 프로세스 종료 후 재실행, 또는 `PORT=<다른포트>` 지정.
- **Playwright 미설치**: 시연 자체엔 불필요 — `npm run test:e2e`(검증용)에서만 필요, `npm install` 후 `npx playwright install` 실행.
- **Ollama 미기동**: `demo:ollama` 없이도 앱은 모의(mock) 에이전트 출력으로 동일하게 동작.
