# D22 — 모델 주권: 외부모델 거부감 해소 · 온프레/로컬 모델 · 컴퓨팅 경제성(전력·TCO) · 하이브리드 시연

## 메타 (프롬프트 밖 — 생성·운영용)
- **도메인**: 은행이 **외부 AI 모델 사용을 꺼리는 문제**를 어떻게 푸나. (A) 외부 LLM 거부감의 실제 사유·해소 스펙트럼, (B) **모델 선택지·주권**(미·중·한 오픈웨이트/상용, 철학·신뢰·국산화 고려), (C) **컴퓨팅·전력·TCO의 수학적 산정**과 JB 자원 적정성, (D) 대회 **시연 논거**(Codex·Claude OAuth + **로컬 AI 모델**을 함께 붙여 "로컬로도 충분히 돌아간다"를 입증).
- **티어**: ★★★
- **심사 항목**: 2(금융 연계·신뢰) · 5(혁신·확장·운영리스크·경제성) + **시연 설득력**
- **실행 배치**: 배치 3 (종합·차별 — D5b·D10·D14·D19 인용)
- **경계 메모**:
  - **D22 ↔ D5b**: 망분리·금융클라우드·생성형AI 허용의 *법·정책 문서* = D5b / *그래서 어떤 모델을 어디서 돌릴지의 선택·경제성·주권* = D22.
  - **D22 ↔ D10**: 인젝션·유출 방어·모델 거버넌스 *보안 기술* = D10 / D22는 *모델 선택·호스팅 형태·비용*.
  - **D22 ↔ D14**: 한국어 금융 도메인 *모델·데이터 품질* = D14 / D22는 *주권·호스팅·경제성* 관점에서 같은 모델군을 평가.
  - **D22 ↔ D19**: 운영 TCO·SLA *전반* = D19 / D22는 **모델 추론 자체의 컴퓨팅·전력·라이선스 경제성**에 집중.
  - **D22 ↔ D20**: 비-LLM 특화모델·도구 = D20 / D22는 *LLM 모델 주권·로컬 호스팅·경제성*.
- **제품 연결**: "PII 비반출 × 승인우선" 하드룰의 **기술적 뒷받침** = 외부 상용 LLM에 원문을 안 보내고도 작동. 시연에서 **Codex·Claude(OAuth)** 같은 강력한 외부 모델과 **로컬/온프레 오픈웨이트 모델**을 **같은 하네스에 동시 연결**해, 민감 작업은 로컬로 처리·"하네스가 좋으면 로컬 모델로도 충분히 돌아간다"를 입증. 핵심 메시지: *모델은 교체 가능한 부품, 가치는 하네스·운영계약에 있다(model-agnostic).*

## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 **금융권 AI 모델 도입 전략·인프라 경제성 리서처**다. 목표는 "은행은 외부 상용 LLM에 고객 데이터를 보내길 꺼린다 — 그렇다면 **무엇을, 어디서, 얼마의 컴퓨팅·전력·비용으로** 돌려야 하며, **한국 은행의 철학·신뢰·국산화**를 고려할 때 어떤 모델 선택지가 합리적인가, 그리고 **로컬/온프레 모델로도 충분히 작동함**을 어떻게 입증하는가"를, **공개 1차 자료·벤치마크·공식 가격/전력 자료**로 정량·구조화하는 것이다. 결과는 "model-agnostic 거버넌드 하네스 + 로컬 우선" 전략을 정당화·시연하는 데 쓰인다.

### 알아내야 할 것 (핵심 질문)

**A. 외부 LLM 거부감 — 실제 사유와 해소 스펙트럼**
1. 은행·금융이 **외부 상용 LLM(클라우드 API) 사용을 꺼리는 구체 사유**: 데이터 유출·주권·규제(망분리·클라우드 가이드)·벤더 종속·기밀·감사. 실제 국내외 금융기관의 입장·발언·정책 근거.
2. **해소 스펙트럼**: 온프레미스 자체호스팅 ↔ 프라이빗/전용 클라우드 ↔ 게이트웨이·프록시(PII 마스킹·비반출) ↔ 데이터 비전송 설계. 각 방식의 트레이드오프(보안·성능·비용·운영부담)와 금융권 채택 사례.

**B. 모델 선택지 · 주권 (미 · 중 · 한)**
3. **로컬/온프레 호스팅에 적합한 오픈웨이트 LLM**의 현황: 미국(예: Llama 등)·중국(예: Qwen·DeepSeek 등)·한국(국산 LLM·금융 특화) 계열을 **라이선스·성능·한국어·크기별로** 비교. (한국어 금융 성능 세부는 D14 참조 — 여기선 *주권·호스팅·라이선스* 축.)
4. **모델 주권·신뢰·국산화 고려**: 금융이 모델 출신국·공급망·데이터 거버넌스·지정학 리스크를 어떻게 따지나(중국계 모델의 규제·신뢰 이슈, 미국계 라이선스·수출통제, 국산 우대). 회사 철학·신뢰·기업간 신뢰 관점에서 합리적 선택 기준.
5. **상용 API vs 오픈웨이트 자체호스팅의 의사결정 프레임**: 어떤 작업을 외부 강력 모델에, 어떤 작업을 로컬에 라우팅하는 **하이브리드/라우팅 전략**의 대표 패턴.

**C. 컴퓨팅 · 전력 · TCO 경제성 (수학적 산정)**
6. **추론 비용의 정량 모델**: 토큰당 비용(상용 API 가격) vs 자체호스팅 비용(GPU 단가·전력·상면·인건·감가). 모델 크기·동시성·처리량(throughput, tokens/s)·지연에 따른 비용 함수. **전력 소비(와트·kWh)와 전력단가**를 포함한 TCO 계산법과 공개 벤치마크 수치.
7. **하드웨어 적정성**: 특정 모델군을 목표 처리량으로 돌리는 데 필요한 GPU/메모리(VRAM)·서버 규모 산정, 양자화·배치·서빙 최적화(추론 효율화)가 비용·전력에 미치는 영향. **"이 회사(은행 규모)에 적절한 자원이 갖춰져 있는가"를 가늠하는 산정 기준**.
8. **손익분기**: 사용량(일/월 토큰·케이스 수)이 얼마를 넘으면 자체호스팅이 상용 API보다 싸지나(break-even). 국내 전력단가·GPU 조달 현실 반영.

**D. 시연 논거 — 하이브리드 + 로컬 입증**
9. **여러 모델을 한 하네스에 연결**(상용+로컬 하이브리드, model-agnostic)한 아키텍처·도구·사례: 동일 에이전트 하네스가 모델을 갈아끼우며 작동하는 설계, **"좋은 하네스 + 작은 로컬 모델"이 실용 성능을 내는** 근거(작업 분해·도구·검증 루프가 모델 한계를 보완). 로컬 소형 모델로 실무가 돌아간 사례·벤치마크.

**E. (범용 덤 유도)** 위 외에 금융 모델 주권·경제성에서 중요한 요소(예: 그린 AI·탄소, 규제기관의 모델 등록·설명요구, 모델 EOL·업데이트 거버넌스)가 있으면 추가하라.

### 출처 규칙 (반드시 준수)
- **공통 5가드**: ① 발견마다 **출처 URL + 원문 짧은 인용 + 발행처·날짜 + 신뢰도(1차/2차/[미검증])**. ② 가격·전력·성능 수치 **2출처 교차**(불가 시 [미검증]). ③ **링크 미접속 시 인용 금지.** ④ **한국 맥락·최신(2024~2026) 우선** — 모델·가격·라이선스는 변동이 크니 **버전·날짜 명시 필수.** ⑤ 의심 시 보수적 [미검증].
- **도메인 가드**: **모델 성능·벤치마크 점수는 출처·일자·평가셋 명시**(리더보드는 변동 — 캡처 일자 필수). **가격·전력·GPU 단가는 공식 가격표·하드웨어 사양·전력단가 고시 1차로만**, 추정은 가정·산식 공개 + [추정] 표기. **라이선스는 원문 확인**(상업적 사용·재배포·출력물 권리). **중국·미국 모델의 규제·수출통제·신뢰 이슈는 단정 금지**, 출처 있는 사실과 의견을 분리.
- **SSOT 우선**: 내부 정본(`_canon.md`)·D5b(규제)·D14(모델성능)와 충돌 시 그쪽 우선.

### 한국 맥락·기간
- 한국 은행·금융지주의 모델 도입·망분리·클라우드 정책, 국내 GPU 조달·전력단가 현실을 최우선. 글로벌은 비교·보강.
- 기간: 모델·가격·라이선스는 **2025~2026 최신** 필수(변동 큼), 산정 방법론은 안정 기법 허용.

### 출력 형식 (이 구조로 답하라)
1. **핵심 발견 요약** — A(거부감·해소)·B(모델 선택·주권)·C(컴퓨팅·전력·TCO)·D(하이브리드 시연)·E(덤) 블록별.
2. **모델 선택지 비교 표** — | 모델군(미/중/한) | 라이선스 | 크기·요구 VRAM | 한국어/금융 적합 | 주권·신뢰 이슈 | 출처·일자 |
3. **거부감 해소 방식 표** — | 방식(온프레/프라이빗/게이트웨이/비전송) | 보안 | 성능 | 비용·운영부담 | 사례 | 출처 |
4. **TCO·전력 산정 표** — | 항목(API 토큰가/GPU/전력kWh/상면/인건) | 단가·출처 | 산식 | 가정·[추정] |
5. **손익분기·시나리오** — 사용량별 상용 API vs 자체호스팅 비용 곡선·break-even(가정·산식 공개).
6. **근거표** — | 주장 | 출처 URL | 발행처·연도·일자 | 신뢰도 | 원문 짧은 인용 |
7. **제품·시연 적용 시사점** — JB LocalGuard OS의 "PII 비반출 × model-agnostic 하네스" 정당화, 시연에서 **Codex·Claude(OAuth)+로컬 모델 하이브리드**로 "로컬로도 충분히 돌아간다"를 입증하는 구체 메시지·구성 (4~7개).
8. **갭 / [미검증]** — 변동 큰 가격·성능의 일자 한계, 한국 직접 사례 희소성, 단정 못 할 지정학·신뢰 이슈.

### 딥리서치 깊이 지시 (Deep Research)
- "모델 나열"이 아니라 **"거부감의 사유 → 해소 방식 → 모델 선택(주권) → 비용·전력 수학 → 하이브리드 시연"의 의사결정 논증**으로 엮어라.
- **C(컴퓨팅·전력·TCO)는 반드시 산식과 단가 출처를 동반**하라 — 숫자 없는 경제성 주장은 [미검증]. 전력(kWh)·GPU 단가·토큰가를 실제 수치로.
- **"좋은 하네스 + 작은 로컬 모델이 실용 성능을 낸다"**의 1차 근거를 우선 파라 — 시연 핵심 메시지다.
- 모델·가격·라이선스는 **버전·일자**를 반드시 달고, 미·중 모델 신뢰·규제 이슈는 **사실/의견 분리**.
- 규제 문서=D5b, 보안 기술=D10, 한국어 모델 성능 세부=D14, 운영 TCO 전반=D19 — **경계에서 멈추고 참조 표기.**
- 제출 전 자체 점검: "TCO에 산식·단가 출처가 있는가 / 모델 수치에 일자가 있는가 / 라이선스 원문 확인했는가 / 신뢰 이슈를 사실·의견 분리했는가 / 로컬-충분 근거가 1차인가."

## Opus 점검 로그 (2패스)

### 패스 1 — 적대적 비판 → 보완
- **비판 ① D5b·D10·D14·D19와 중복.** 규제·보안·한국어모델·TCO가 기존 도메인과 겹친다.
  - **보완**: 메타에 5경계 규칙 — *규제 문서=D5b / 보안기술=D10 / 모델 성능=D14 / 운영TCO 전반=D19*. D22 고유 축 = *거부감 해소 + 모델 주권 + 추론 컴퓨팅·전력 경제성 + 하이브리드 시연*으로 좁힘.
- **비판 ② 경제성 주장이 숫자 없이 헐거울 위험.** "자체호스팅이 싸다" 류는 산식·단가 없으면 무의미.
  - **보완**: C6~C8을 토큰가·GPU 단가·전력 kWh·throughput의 **정량 모델·break-even**으로 못 박고, 출력 4·5 표에 "산식·단가 출처·[추정]"을 강제. 깊이지시에 "숫자 없는 경제성=[미검증]".
- **비판 ③ 모델·가격 변동성 → 즉시 노후화.** 리더보드·가격은 몇 주면 바뀐다.
  - **보완**: 도메인 가드·기간에 "버전·일자 명시 필수, 2025~2026 최신"을 박고 출력 2 표에 "출처·일자" 열을 요구.

### 패스 2 — 재비판 → 보완
- **재비판 ① 미·중 모델 신뢰·지정학 단정 위험.** 중국계 모델 "위험"·미국계 "안전" 단정은 편향·반론 자초.
  - **보완**: B4·도메인 가드·깊이지시에 "**사실과 의견 분리**, 규제·수출통제·신뢰는 출처 있는 사실만, 추정 [미검증]"을 강제.
- **재비판 ② 시연 메시지가 자랑으로 흐를 위험.** "로컬로도 된다"가 근거 없는 호언이면 심사 역효과.
