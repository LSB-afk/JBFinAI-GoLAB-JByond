## 딥리서치 프롬프트 (이 아래 통째로 붙여넣기)

당신은 **금융 AI 도입의 비용·편익·리스크를 정량화하는 계량경제·재무모델링 전문가**다. 목표는 "지역은행이 거버넌드 AI 에이전트 콘솔을 도입했을 때의 **경제성·리스크를 방어 가능한 수학 모델**로 세우는 것"이다. 막연한 주장 금지 — 모든 결론은 **변수 정의 · 산식 · 단위 · 출처 있는 파라미터 · 명시된 가정 · 민감도**로 뒷받침하라. 출력의 모든 수식은 **LaTeX**(`$...$` / `$$...$$`)로 표기하라(이미지·말풍선 금지).

### ⚠️ 0. 모델 검증·정정 먼저 (가장 중요)
**아래 M1~M4에 제시된 산식은 "확정 스펙"이 아니라 검증·정정 대상 초안(strawman)이다.** 너의 첫 임무는 모델을 채우는 게 아니라 **이 산식들이 각 분야의 표준·문헌과 맞는지 검증하고, 틀리거나 과단순하거나 더 적합한 정식 모델이 있으면 교체하는 것**이다. 구체적으로:
- 각 산식을 **권위 있는 1차/표준 출처**(재무: NPV/IRR/실물옵션 교과서·CFA류; 신용리스크: Basel IRB의 EL/PD/LGD/EAD 정의; 대기행렬: M/M/c·M/G/c 등 OR 표준; LLM 추론원가: 공식 서빙·벤치마크 문헌; 인건·전력단가: 공식 통계)와 **대조**하라.
- 초안이 **틀렸거나(예: 변수 누락·단위 오류·잘못된 함수형), 과단순하거나(예: 단일 대기행렬이 부적합), 분야 표준과 다르면 정정·교체**하고, **무엇을 왜 바꿨는지** 근거·출처와 함께 보고하라.
- 내가 놓친 **추가 항목**(예: 모델 재학습·드리프트 비용, 양자화에 따른 품질저하, 오탐 처리비용, 변화관리·교육비, 실물옵션적 단계도입 가치)이 표준 모델에 필요하면 **추가**하라.
- 결론: 너의 최종 모델은 **초안이 아니라, 문헌으로 검증·정정된 모델**이어야 한다. 초안을 무비판 수용하지 말 것.

전체를 **5개 서브모델 + 1개 통합 모델**로 구성하라. 각 서브모델은 ① 변수표(기호·의미·단위·기준값·출처/[추정]·합리적 범위) ② 산식(LaTeX) ③ JB 규모 가정의 **계산 예시(worked example)** ④ **민감도**(어떤 변수가 결과를 지배하나)를 포함한다.

### M1. AI 추론·인프라 TCO 와 손익분기 (입력: D22)
- **상용 API 비용**: $C_{api} = T_{month}\cdot p_{token}$ (월 토큰량 × 토큰단가; in/out 분리). 최신 상용 LLM **토큰 단가**를 모델·일자와 함께 수집.
- **자체호스팅 비용(월)**: $C_{self} = \dfrac{Capex_{GPU}}{L} + E\cdot p_{kWh} + OPEX$ — GPU 자본지출/수명 + 전력(에너지 $E$[kWh] × 전력단가) + 운영·상면·인건. 전력 $E = P_{watt}\cdot u\cdot h /1000$ (소비전력·가동률·시간).
- **처리량 제약**: 목표 처리량(토큰/s 또는 케이스/일)을 내는 데 필요한 **GPU 대수·VRAM**을 모델 크기·서빙효율(배치·양자화)로 산정.
- **손익분기**: $C_{api}=C_{self}$ 가 되는 사용량 $T^{*}$. 사용량별 비용 곡선과 "언제 자체호스팅이 싸지나".
- 수집 파라미터(출처 필수): 상용 토큰 단가(일자), 국내 **전력단가(원/kWh)**, GPU 시간당/구매 단가·소비전력, 대표 오픈웨이트 모델 throughput 벤치마크.

### M2. 업무 절감·처리량 베이스라인 (입력: D16·D8)
- **연간 절감액**: $S = \sum_i N_i\cdot \Delta T_i\cdot C_i \cdot R$ — 과제 $i$별 (건수 $N$ × 건당 단축시간 $\Delta T$ × 시간당 인건원가 $C$) × **실현율** $R$(0–1, 채택·자동화 한계 반영).
- **처리량/인원(대기행렬)**: 케이스 도착률 $\lambda$, 처리율 $\mu$, 인원 $c$ 의 $M/M/c$ 로 대기시간·SLA·필요 인원 산정. AI 도입이 $\mu$(처리율)를 올리는 효과를 모델화.
- 수집: 한국 금융 사무직 **시간당 인건원가**, 과제별 처리시간 벤치마크(D8 효과크기·신뢰구간 활용), 실현율의 합리적 범위.

### M3. ROI · Payback · NPV (입력: D+a)
- $ROI = \dfrac{B - C}{C}$, $Payback = \dfrac{Capex_0}{B_{annual}-O_{annual}}$, $NPV=\sum_{t=0}^{T}\dfrac{CF_t}{(1+r)^t}$, IRR.
- 편익 $B$ = M2 절감 + 리스크 회피(M4) + 매출/유지 효과(있으면). 비용 $C$ = M1 TCO + 구축·라이선스·운영.
- **할인율 $r$**·기간 $T$ 가정 명시. 보수/기준/낙관 3 시나리오.

### M4. 리스크 보정 · 시뮬레이션 (입력: D21)
- **기대손실(신용 표준)**: $EL = PD\cdot LGD\cdot EAD$. 사기·전세 등은 $EL=p\cdot L$ (사건확률 × 피해액). AI가 $PD$/탐지율/오탐을 어떻게 바꾸나로 **리스크 회피 편익**을 산출.
- **몬테카를로**: 입력(채택률·단가·절감시간·확률)을 분포로 두고 시뮬레이션 → 편익·NPV의 **분포(P10/P50/P90)** 와 **VaR**. 단일 점추정의 과신을 방지.
- **시나리오 시뮬(행동 전 검증)**: 외부변수(예: 농가 여신의 기상·작황·시세) 분포를 넣어 결과 분포·하방리스크를 보여주는 "simulation-before-action" 템플릿.
- 수집: 대표 PD/LGD·사기 평균 피해액 등은 D21/D1을 인용(없으면 출처와 함께, 단정 금지·[추정]).

### 통합. 위험조정 비즈니스 케이스
- $\text{위험조정 순편익}_t = B_t - EL_t - TCO_t$ → NPV/IRR. M1~M4를 **한 모델로 연결**(한 변수 변화가 전체에 어떻게 전파되나).
- **민감도/토네이도**: 결과(NPV)를 가장 크게 흔드는 상위 변수 5개. 가정이 깨지면 결론이 어디서 뒤집히나(손익분기 임계).

### 출처·검증 규칙 (반드시 준수)
- **공통 5가드**: ① 파라미터마다 **출처 URL + 발행처·일자 + 신뢰도(1차/2차/[추정])**, 추정은 산정 근거 명시. ② 단가·전력·성능은 **2출처 교차**(불가 시 [추정]). ③ 링크 미접속 시 인용 금지. ④ **한국 맥락·최신(2025~2026)** 우선 — 단가·토큰가·전력단가는 **일자 필수**(변동 큼). ⑤ 의심 시 보수적 범위.
- **모델 가드(강)**: ⓐ 모든 수식 **LaTeX**, 단위 명시. ⓑ **모든 점추정에 범위(min/기준/max)** 동반 — 단일 숫자 금지. ⓒ 가정은 별도 "가정표"에 모아 추적 가능하게. ⓓ "AI가 X% 개선" 류는 D8 효과크기·신뢰구간 또는 출처로만, 없으면 [추정]. ⓔ 결과는 **민감도와 함께**(어디서 결론이 뒤집히는지).
- **SSOT 우선**: `_canon.md`·D16·D+a·D22·D21 회수본과 충돌 시 그쪽 1차 수치 우선.

### 패스 2 — 재비판 → 보완
- **재비판 ① 회수 전이라 입력 공백.** D16·D22·D21·D+a가 아직 미회수면 모델이 빈다.
  - **보완**: 배치 메모에 "미회수 시 출처 있는 외부 파라미터+가정으로 1차 모델 → 회수 후 보정", 출력 8 갭에 "회수 대기 입력 명시"를 요구. 모델 **구조**는 지금 완성, 값은 갱신.
- **재비판 ② 수식이 산문에 묻힐 위험(워드 수식 이미지 사고 재발).** AI·옵시디언이 못 읽으면 무용.
  - **보완**: 모델 가드 ⓐ·깊이지시에 "**모든 수식 LaTeX, 이미지 금지**"를 반복 박음(이전 D16 워드 수식 이미지→LaTeX 변환 교훈 반영).
- **재비판 ③ 편익 이중계산·경계 누수 위험.** 절감(M2)·리스크회피(M4)·매출효과를 겹쳐 세면 ROI 거품.
  - **보완**: M3 편익 정의에 "M2 + M4 + (있으면)매출, **중복 배제**"를 명시하고 통합 산식에서 $B_t,\,EL_t,\,TCO_t$ 를 분리해 이중계산을 구조적으로 차단.

### 패스 3 — 사용자 비판 반영(2026-07-01): "내가 세운 산식이 틀릴 수 있다"
- **비판(사용자)**: 프롬프트에 산식을 확정해 박으면, Gemini가 **틀린 골격을 성실히 채우기만** 하고 리서치 중 발견한 더 정확한 모델로 못 바꾼다. 작성자(Opus)의 모델이 표준과 다를 수 있음.
  - **보완**: 산식을 **검증·정정 대상 초안(strawman)으로 강등**. 프롬프트 맨 앞에 "**0. 모델 검증·정정 먼저**"를 신설 — 각 산식을 Basel IRB(신용)·OR 대기행렬·재무(NPV/IRR/실물옵션)·LLM 서빙 문헌 등 **표준과 대조→정정/교체**하고 변경 이유·출처를 보고할 의무. 출력 0에 "모델 검증·정정 로그" 표(초안|대조결과|최종 산식|이유|출처) 추가. 깊이지시 최상단에 "초안 무비판 수용 금지, 정식 모델 교체가 핵심 가치". 누락 가능 항목(재학습·드리프트 비용·오탐 처리비·변화관리·실물옵션 단계도입 가치) 추가 검토도 지시.
