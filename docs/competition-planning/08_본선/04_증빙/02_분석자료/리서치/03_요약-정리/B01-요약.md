# B1 요약 — 오픈소스 롤모델은 StackStorm·Temporal, 디자인 기준은 paperclip+Carbon이다

## ⚡ 30초 핵심
기능 롤모델은 `StackStorm`와 `Temporal` 조합이 가장 현실적이다.  
디자인은 사용자 제공 `paperclip(MIT)`를 1차 본보기로 두고, `Carbon` 토큰 규율과 `Tremor` 대시보드 블록을 보조로 쓴다.  
`TheHive`, `Windmill`, `n8n`, `Midday`, `Twenty`, `Grafana`는 AGPL/GPL/fair-code 리스크가 있어 소스 lift 없이 클린룸 재구현만 허용한다.

## 🎯 바로 쓸 것 (에이전트·제품 즉시 활용)
- `Approval` 상태를 `paused/pending/approved/rejected/expired`로 고정하고, `core.ask`식 승인 게이트와 JSON schema 응답 구조를 `StackStorm` 패턴으로 설계한다.
- `AgentRun` 계약은 `Temporal`식으로 잡는다. `signals/approve`, append-only `events`, case detail 우측 타임라인을 기본 shape로 둔다.
- 최소 감사 필드는 `triggered_by`, `started_at`, `ended_at`, `input_snapshot`, `output_snapshot`, `approval_snapshot`로 정리한다.
- 프론트는 `paperclip`의 `Case/Agent/Approval/Audit` IA를 기준으로 맞추고, `Carbon` productive typography·table density·theme token을 CSS 변수로 이식한다.
- 대시보드 블록은 `Tremor`식 `KPI strip + chart card + filter bar + empty/error/loading state` 조합으로 쪼개어 HTML 템플릿으로 재구현한다.
- permissive 라이선스여도 React/TS 런타임은 들여오지 않는다. 재사용 단위는 `token + IA + state machine + API contract`까지만 자른다.

## 🙋 전문가에게 묻는다면
- **Q. 며칠 안에 실제 데모까지 가능한 롤모델은 뭔가?** A. `StackStorm`식 승인 게이트와 `Carbon`식 토큰 체계다. 엔진 전체가 아니라 승인 큐, 상태 전이, 실행 이력, 테이블 규율을 먼저 가져온다.
- **Q. `Temporal`은 지금 바로 붙이나?** A. 아니다. 지금은 `Signal + Event History` 계약만 차용한다. 클러스터·워커 도입은 며칠짜리 작업이 아니다.
- **Q. 왜 `TheHive`가 아니라 `StackStorm`인가?** A. `TheHive`는 `Case/Evidence/Audit`는 강하지만 공개 OSS 계보가 archive 상태고 승인 게이트가 약하다. 이번 범위에는 `StackStorm`가 더 가깝다.
- **Q. AGPL 계열은 완전히 배제하나?** A. 소스, CSS, 컴포넌트 lift는 배제한다. 화면 인상과 정보구조만 보고 클린룸으로 다시 짠다.
- **Q. 디자인 기준은 결국 뭐로 못 박나?** A. 1순위는 `paperclip`, 보조는 `Carbon`과 `Tremor`다. 원문도 이 위상 갱신을 명시했다.

## ⚠️ 주의·한계
- `Midday`, `Twenty`는 금융·CRM 톤 참고용이다. 세부 토큰과 컴포넌트 규칙은 원문에서도 `[미검증]`로 남겼다.
- `Temporal`, `Trigger.dev`, `Windmill`, `n8n`이 주는 런타임 성질은 화면 모사와 다르다. 데모 UI와 엔진 완성도를 섞어 말하면 안 된다.
- permissive 라이선스라도 바닐라 JS 이식성이 자동으로 확보되지는 않는다. 안전한 것과 쉬운 것은 별개다.
