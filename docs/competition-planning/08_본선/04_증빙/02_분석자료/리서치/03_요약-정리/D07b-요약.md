# D7b 요약 — 상용 대체데이터는 사도 되고, 원문은 밖으로 보내면 안 된다

## ⚡ 30초 핵심
CB·카드·사기방지 상용 데이터는 RM 고도화에 유용하지만, 라이선스와 신용정보법 때문에 원본 응답을 외부 LLM에 그대로 태우면 안 된다.  
안전한 방식은 로컬 게이트웨이에서 원본을 받아 `구간화`, `등급화`, `플래그화`한 뒤 익명 메타데이터만 LLM에 넣는 구조다.  
적합도는 `KB부동산 시세=적합`, `NICE/KCB/CRETOP/카드매출/더치트=조건부 적합`, `금융보안원 ASAP=부적합`으로 정리된다.

## 🎯 바로 쓸 것 (에이전트·제품 즉시 활용)
- 데이터 버스는 `공공 데이터 기본 경로`와 `상용·대체데이터 확장 경로`로 분리한다.
- NICE BizAPI, KCB BIZDATA, CRETOP은 원본 수치 대신 `신용 리스크 세그먼트`, `휴폐업 이력`, `부도확률 구간` 같은 요약 지표만 LLM에 넣는다.
- 카드매출 데이터는 점포 상세 매출액을 외부로 보내지 말고 `매출 하락율 단계`, `권역 평균 대비 하락`, `폐업 경고 수준`으로 치환한다.
- 더치트는 전화번호·계좌번호 원문 조회를 로컬 서버에서만 처리하고, LLM에는 `사기 피해 등록 유무` 플래그만 넘긴다.
- ASAP 위협지표는 외부 프런티어 LLM과 섞지 말고 은행 내부 FDS·룰엔진 전용 피드로 격리한다.
- 전세 보강용 상용 데이터가 필요하면 KB부동산 시세 API를 우선 후보로 두고, 주소는 단지·권역 통계 수준으로 비식별화한다.

## 🙋 전문가에게 묻는다면
- **Q. CB 데이터를 사오면 외부 LLM에 바로 붙여도 되나?** A. 안 된다. 약관과 신용정보법 때문에 로컬 비식별·추상화가 먼저다.
- **Q. 소상공인 매출 둔화 신호는 어떻게 주입하나?** A. 카드사 원시 매출액 대신 `Sales Drop of 24.5% compared to quarterly average` 같은 요약 지표로 바꿔 넣는다.
- **Q. 상용 데이터 중 가장 다루기 쉬운 건 뭔가?** A. 원문은 KB부동산 시세를 `적합`으로 보고, NICE/KCB/CRETOP·카드매출·더치트는 `조건부 적합`으로 봤다.
- **Q. 더치트나 ASAP를 같은 방식으로 쓰면 되나?** A. 아니다. 더치트는 플래그화하면 조건부 가능하지만, ASAP는 금융회사 내부 FDS 한정이라 외부 LLM 경로엔 부적합이다.
- **Q. 비용이 공개된 상품도 있나?** A. CRETOP은 `10건 7,000원 ~ 3,000건 1,260,000원`, 더치트는 `월 600,000원`, `20,000건 포함`, 초과 `건당 10~30원`이 원문에 있다.

## ⚠️ 주의·한계
- NICE, KCB, BC카드 같은 핵심 상용 API는 대체로 공식 단가표가 비공개라 영업 협의가 필요하다.
- 이통 3사 대안 신용 스코어 API 규약과 은행 전용망 연동 허용 범위는 미검증이다.
- 신한카드 상권·매출 데이터의 사설 AI 콘솔 상시 동기화 허용 범위도 계약서 확인 전까지 미검증이다.
