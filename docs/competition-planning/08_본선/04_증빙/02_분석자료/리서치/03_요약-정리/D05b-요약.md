# D5b 요약 — 망분리는 남아 있고 외부 생성형 AI는 예외 트랙으로 쓴다

## ⚡ 30초 핵심
2026년 6월 기준 금융권 기본 규칙은 여전히 망분리다.  
다만 `SaaS 예외`, `혁신금융서비스`, `R&D 예외`가 열리면서 외부 생성형 AI와 클라우드를 쓰는 길이 생겼다.  
가장 안전한 구조는 외부 LLM은 SaaS·샌드박스 경로로 쓰고, 고객 PII·개인신용정보는 외부로 내보내지 않는 방식이다.

## 🎯 바로 쓸 것 (에이전트·제품 즉시 활용)
- 정책 설명은 `망분리 폐지`가 아니라 `망분리 원칙 + 예외 + 보안통제 + 결과책임`으로 표현한다.
- 외부 프런티어 LLM은 `SaaS형 서비스`로 정의하고, 내부 업무망 사용 시 정보보호위원회 보고, 반기 1회 평가, 단말 보호대책을 전제로 둔다.
- 외부 프롬프트·로그·툴콜에서는 고객명, 주민번호, 계좌식별값, 개인신용정보를 전부 제거한다.
- 가명정보를 외부 LLM으로 보내는 경우도 일반 SaaS 예외로 끝내지 말고 `혁신금융서비스 지정` 트랙으로 본다.
- 모델 라우팅은 `비식별·비신용정보형 작업은 외부 LLM`, `민감·식별·신용 데이터는 내부 모델·온프레미스`로 나눈다.
- 운영 문서에는 `제3자 리스크 관리`, `모델 변경 사전 점검`, `R&D망과 운영망 분리`를 필수 통제로 넣는다.

## 🙋 전문가에게 묻는다면
- **Q. 지금 금융권에서 외부 생성형 AI를 아예 못 쓰나?** A. 아니다. 샌드박스와 SaaS 예외를 통해 이미 지정·운영 사례가 나왔다.
- **Q. 내부 업무망에서 SaaS형 LLM을 바로 붙여도 되나?** A. 조건부다. 보안 규율을 지켜야 하고, 고유식별정보·개인신용정보 처리에는 일반 예외가 적용되지 않는다.
- **Q. 가명정보를 보내면 일반 SaaS 예외로 충분한가?** A. 아니다. 원문은 가명정보도 별도 혁신금융서비스 경로가 더 안전한 해석이라고 본다.
- **Q. 제품에서 제일 중요한 통제 포인트는 뭔가?** A. 무엇을 외부로 내보내지 않느냐다. 외부 반출 금지 데이터 범위를 먼저 고정해야 한다.
- **Q. 외부 모델을 계속 바꿔도 되나?** A. 경미 변경이라도 보안영향 점검과 지정 서비스 변경 절차를 따라야 한다.

## ⚠️ 주의·한계
- `MLS`나 `다층보안체계`는 이번 조사에서 공식 원문 용어로 직접 확인되지 않았다.
- 2026-04-20 시행세칙 본문 전체는 웹 렌더링 한계가 있어 일부 직접 인용이 미검증이다.
- 금융보안원 2025 클라우드 가이드 PDF 전문은 추가 확보 후 세부 절차를 다시 검증하는 편이 안전하다.
