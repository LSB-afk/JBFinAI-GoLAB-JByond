# D5a 요약 — 외부 LLM은 금지보다 가명정보·위탁 통제로 다뤄야 한다

## ⚡ 30초 핵심
금융권 PII 비반출 설계의 법적 앵커는 `신용정보법`이고 `PIPA`는 보충 규범이다.  
외부 LLM에 데이터를 보내려면 원본 PII는 내부에만 두고, 외부에는 가명·토큰화된 최소 정보만 보내야 한다.  
토큰↔원본 키 분리보관, 출력 재식별 차단, 처리기록 3년 보존은 권고가 아니라 법상 통제다.

## 🎯 바로 쓸 것 (에이전트·제품 즉시 활용)
- 제품 설명 문구는 `원본 개인신용정보는 은행 내부 유지`, `외부 LLM에는 가명·토큰화 최소정보만 라우팅`, `추가정보는 분리·암호화·접근통제`, `출력 재식별 차단`, `처리기록 3년 보존`으로 고정한다.
- 데이터 등급은 최소 `원본 개인신용정보`, `가명정보`, `익명정보`, `AI 입력 불가`, `AI 입력 가능` 5단계로 나눈다.
- `토큰↔원본 키 분리보관`은 HSM 또는 별도 암호화 저장소 전제로 설계하고, 가명정보 취급자와 추가정보 접근권한을 분리한다.
- 외부 LLM 라우팅은 `처리위탁 계약`, `목적 외 사용 금지`, `재위탁 통제`, `로그 감사 가능`, `학습·재판매 금지` 조건을 계약과 시스템에 같이 건다.
- 출력단에는 주민번호, 계좌식별자, 주소 조합, 희소 속성 조합을 스캔하는 반출 차단기를 둔다.
- 감사원장에는 `가명처리 규칙`, `외부모델 라우트`, `승인`, `접근자`, `처리 목적·방법·일시`, `스캔 결과`를 남기고 3년 이상 보존한다.

## 🙋 전문가에게 묻는다면
- **Q. 금융권 LLM 설계에서 어느 법을 먼저 보나?** A. 신용정보법이 특별법으로 우선이고, PIPA는 보충 적용이다.
- **Q. 토큰화만 하면 외부 LLM 전송이 자동으로 적법한가?** A. 아니다. 추가정보 분리보관, 외부 사업자의 재결합 불가, 출력 재식별 차단, 위탁 구조가 같이 있어야 한다.
- **Q. 외부 사업자가 입력 데이터를 자기 모델 학습에 쓰면 어떤 문제인가?** A. 처리위탁이 아니라 독립 이용·제3자 제공 문제로 커진다.
- **Q. 로그는 어느 수준까지 남겨야 하나?** A. 누가 어떤 데이터를 어떤 가명처리 규칙과 외부모델 경로로 처리했고 어떤 차단·승인이 있었는지까지 남겨야 한다.
- **Q. 위반하면 제재가 어느 정도인가?** A. 재식별은 5년 이하 징역 또는 5천만원 이하 벌금까지 걸리고, 과태료와 매출액 기준 과징금도 중첩될 수 있다.

## ⚠️ 주의·한계
- 공개 제재사례는 많지만 `신용정보법 §40조의2 재식별 위반`을 정면 처분한 사례는 이번 조사에서 확인되지 않았다.
- PIPA `§28조의5` 문언 변화의 정확한 개정 시점은 부분 미검증이다.
- 2026년판 `금융분야 가명·익명처리 안내서`는 예고만 확인됐고, 공개 게시본은 2022.1 기준이다.
