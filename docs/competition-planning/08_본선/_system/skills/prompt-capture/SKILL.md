# prompt-capture

> 세션의 **사용자 프롬프트 원문**을 분기코드로 분류해 [[프롬프트-로그]]에 누적한다. 토큰·도구·시간은 Stop훅(session-telemetry)이 자동이지만, **프롬프트 텍스트+분기 분류는 AI 판단**이 필요해 이 스킬이 담당한다.

## 실행

```bash
# 이 세션의 사용자 프롬프트만 결정론적으로 추출 (분류 X)
node 08_본선/_system/skills/prompt-capture/extract-prompts.mjs
# 특정 트랜스크립트 지정 / 회귀검증
node .../extract-prompts.mjs <transcript.jsonl>
node .../extract-prompts.mjs --self-test
```

## 수행 작업 (AI 단계)

1. **추출** — `extract-prompts.mjs` 실행 → 이 세션 사용자 프롬프트를 시간순으로 받는다(명령 래퍼·툴결과·시스템리마인더는 이미 제외됨).
2. **중복 제거** — [[프롬프트-로그]]에 이미 기록된 프롬프트(같은 원문)는 **건너뛴다**(멱등). 마지막 항목 ID 이후의 신규 프롬프트만 대상.
3. **분기 분류** — 각 신규 프롬프트를 분기코드에 배정:
   - 메인 파일 `## 최신 활성 구간`의 **마지막 항목 코드**를 확인해 이어서 번호(예: `V-21` 다음은 `V-22`).
   - **주제가 크게 갈라지면** 새 코드를 쓸 수 있으나, 신규 `## [코드]` 섹션은 만들지 않는다 — 로그 로테이션(AGENTS §4) 이후 메인에는 분기별 `## [코드]` 섹션이 없고 `## 최신 활성 구간` 하나만 있다. 과거 `## [S]/[R]/[T]/[C]/[P]/[D]` 섹션은 전부 `_아카이브/`로 이월됨(분기표 자체는 로그 서두에 남아 코드 의미 설명용으로만 유지).
4. **항목 작성** — `#### 코드번호 · YYYY-MM-DD[ HH:MM KST] · 한 줄 제목` → `> 원문 인용` → `> → 산출 요약`(무엇을 했는지·결과물 위키링크·핵심 단서). 시각은 트랜스크립트 타임스탬프(KST), 불확실하면 날짜만.
5. **append** — 메인 파일 `## 최신 활성 구간` 섹션 끝에 추가. 메인이 300줄을 넘으면 로테이션(AGENTS §4) 수행 — 오래된 항목을 `_아카이브/<로그명>-<기간>.md`로 이월(삭제 금지).

## 규율

- **원문 보존** — 사용자 표현·오타를 그대로 인용(`>`). 요약은 `→` 줄에만.
- **소급 보강** — 세션 중 누락분 발견 시 시간 위치에 맞춰 소급 삽입(R10 선례).
- **PII** — 결제·실명·연락처가 프롬프트에 있으면 [[pii-governance-validator]] 규칙대로 마스킹 후 기록.
- 도달성: 이 로그는 이미 `_04_증빙_MOC` 자식 → 항목 추가만으로 충분(새 파일 아님).

## 트리거 (AGENTS §4-A)
세션 체크포인트·종료, "프롬프트 기록/남겨라" 요청, [[harness-sync]] 루틴 중 → 자동 시행.
