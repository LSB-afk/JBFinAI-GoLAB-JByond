# 디자인 토큰 업그레이드 설계안 — JB LocalGuard OS 콘솔 (2026-07-04)

> 목적: 복잡한 정보 밀도를 가진 콘솔에서도 "판단 지원" UX가 유지되도록 디자인 철학을 명문화하고, 여러 곳에 흩어진 디자인 소스를 하나의 토큰 체계로 통합한다. 반복 수정 → 잠금(lock) → 프로토타입 개발 착수 순서로 진행. 이 문서는 [[초안]] 단계이며, 세션 중 사용자가 자리를 비워 §6 충돌 3항목은 **추천안을 기본값으로 채택**했다 — 김민주·이승보·김주용 확인 후 뒤집힐 수 있다. **사용자 복귀 후 §2 원칙 0·5, §3 프로토타입 대상을 사용자 직접 지시로 갱신**([[프롬프트-로그]] V-10).

## 0-1. 사용자 복귀 후 핵심 방향 (직접 지시, 재확인 불요)

- **밀도 우선**: "직원들을 위한 업무 프로그램이라 엄청 예쁠 필요는 없다. 정보의 위계·분류·가독성이 중요하다. 간격은 너무 넓으면 안 된다. 사용 목적에 맞되 때깔은 JB와 어울려야 한다." → 기존 design-system.md §1 원칙4의 "넉넉한 여백으로 IR 대시보드급 밀도" 문구는 이 지시와 어긋나므로 **개정 대상**(§2-0).
- **실제 프로토타입 = paperclip 기반 자체 프로그램**: "실제 개발은 paperclip의 커스텀 프로그램을 우리는 만들 것" — 뼈대·시각정보 표시·에이전트 LLM이 콘솔로 도는 방식은 paperclip의 디자인과 언어를 가져와 JB화한다. 기능 방향은 승보 JB_project2(4-하네스 역할 콘솔)를 따라가되 "아직은 많이 부족" — 즉 **paperclip=디자인/구조 도너, JB_project2=기능 방향 참고(그대로 베끼지 않음)**.
- **JB_project2 병행 개발 확인**: 원격 PR [github.com/LSB-afk/JB_project2#1](https://github.com/LSB-afk/JB_project2/pull/1)(`feat/live-agent-ccl-financial`, 로컬 `_vendor/JB_project2/` 커밋 `678be10`)에서 `llmClient.js`(LLM 호출 seam)·`cclConsole.app.js`(에이전트 실행+배지 UI)가 실시간으로 추가되는 중 — 에이전트 LLM이 실제로 콘솔에 배지·상태로 드러나는 패턴의 살아있는 예시로, 컴포넌트 상태색 설계(§1 Case FSM/AgentRun) 시 참고할 실물.

## 2. 원칙 (0·5는 사용자 직접 지시로 확정, 1~4는 추천안)

0. **밀도·위계·가독성 우선(확정, 사용자 직접 지시)**: 이 콘솔은 IR 대시보드가 아니라 **직원용 업무 프로그램**이다 — "엄청 예쁠 필요 없음". 정보 위계·분류·스캔 가독성이 최우선이고, 카드 간격·여백은 타이트하게(현 design-system.md §1의 "넉넉한 여백/32px 카드/IR 대시보드급 밀도" 문구는 개정 대상 — §5 개정 시 spacing 스케일을 좁은 쪽으로 재설정). 그러나 색·타이포의 "때깔"은 JB 브랜드와 어울려야 한다 — 밀도는 올리되 브랜드 아이덴티티(네이비·블루)는 유지. 현재 브랜치 `ui-density-improvements`의 취지와 일치.
1. **기존 [확정] 항목은 (밀도 원칙 제외) 유지한다**: 라이트 테마, 3열 셸, SUIT Variable, L0~L4. 2026-07-02에 다크/8px 초안을 명시적으로 폐기한 이력이 있고, designer 역할의 "제안→승인" 결정권 범위(색상·타이포·레이아웃 변경) 밖에서 이를 세션 혼자 힘으로 뒤집지 않는다.
2. **Figma 산출물은 "흡수 대상"으로 취급한다**: 지금까지 없던 것(Case FSM 5상태, 컴포넌트 8종 명세, PII UI 규칙, 인터랙션 계약)은 그대로 design-system.md에 신규 섹션으로 편입한다.
3. **색상값은 재매핑, 그대로 채택하지 않는다**: Figma의 4단계 리스크 팔레트(Critical/High/Medium/Low)는 기존 L0~L4 5단계 네이밍에 맞춰 L1~L4로 재매핑(L0은 기존 중립 회색 유지)하는 안을 기본값으로 제안 — 리스크 레벨 수 자체를 바꾸는 것은 §6 결정 대기.
4. **Enter-First 철학을 원칙 문서의 근거로 승격**: 기존 §1 "일은 에이전트, 사람은 눈만"을 승보 UX 분석의 HCI 근거(recognition-over-recall·Hick's Law·Fitts's Law·Progressive Disclosure)로 보강하고, 리스크 방어 규칙(고위험은 Enter만으로 진행 불가 등)을 §6 접근성/컴포넌트 절에 명시한다.
5. **paperclip = 디자인·구조 도너, JB_project2 = 기능 방향 참고(확정, 사용자 직접 지시)**: 실제 프로토타입은 paperclip의 뼈대·시각정보 레이아웃·"에이전트 LLM이 콘솔로 도는" 상호작용 언어를 가져와 JB 토큰으로 재피부화(reskin)한다. 승보 JB_project2(4-하네스 역할 콘솔)는 **기능 방향**만 따라가고 시각 디자인은 따라가지 않는다 — "아직은 많이 부족"하다는 사용자 평가 그대로 반영. 이 원칙은 §3 산출물 구조·프로토타입 대상 결정의 근거다.
