# 역할 기반 금융 AI 업무 콘솔 발표자료 정리

## 0. 제목

**역할 기반 금융 AI 업무 콘솔**

담당자의 판단을 보조하고, 운영자가 AI 협업을 감독하는 엔터프라이즈 콘솔

부제 후보:

- **계열사 x 역할 x 케이스 기반 AI 업무 자동화 플랫폼**
- **금융 조직의 역할별 판단 업무를 AI와 함께 처리하는 운영 콘솔**
- **AI 추천, 사람 승인, 감사 로그를 하나로 묶은 금융 업무 콘솔**

## 1. Summary

본 솔루션은 금융사 내부 담당자가 반복적으로 처리하는 심사, 상담, 이상거래, 전세보호 등 복잡한 업무를 **역할 중심 AI 콘솔**로 통합한다.

기존 업무는 도메인별 시스템이 흩어져 있어 담당자가 여러 화면과 문서를 오가며 판단해야 했다. 본 솔루션은 담당자 역할을 기준으로 업무를 재구성하고, 각 역할이 처리하는 케이스에 대해 AI가 근거 수집, 위험 분석, 추천안 생성, 승인 기록을 지원한다.

핵심은 AI가 단독으로 결정하는 완전 자동화가 아니라, **담당자가 AI의 근거를 확인하고 승인하는 human-in-the-loop 구조**다. 운영자는 역할별 업무 흐름과 AI 협업 상태를 하나의 콘솔에서 감독할 수 있다.

## 2. 문제 정의(Problem Definition)

현재 금융 업무의 문제는 단순히 데이터가 부족한 것이 아니라, **역할과 도메인이 섞여 있어 업무 판단 흐름이 불명확하다**는 점이다.

예를 들어 `RM`, `여신`, `전세`, `보이스피싱`이 같은 수준의 메뉴처럼 섞이면 사용자는 자신이 어떤 역할로 어떤 케이스를 처리하는지 명확히 파악하기 어렵다.

정리하면 문제는 세 가지다.

1. 담당자 역할과 업무 도메인이 한 화면 안에서 혼재된다.
2. 케이스 판단에 필요한 근거가 여러 시스템과 문서에 분산된다.
3. AI 추천 결과를 사람이 검토하고 승인한 이력을 남기기 어렵다.

따라서 본 솔루션은 업무를 **도메인별 앱**이 아니라 **담당자 역할별 콘솔**로 재구성한다.

## 3. 제안 솔루션 개요(Solution Overview)

본 솔루션은 **계열사 x 역할**을 1차 조직축으로 삼고, 각 역할 안에서 도메인 케이스를 처리하는 AI 업무 콘솔이다.

기본 구조:

```text
계열사 -> 역할 -> 케이스 -> AI 분석 -> 근거 확인 -> 담당자 승인 -> 감사 로그
```

예시 구조:

| 계열사 | 역할 | 처리 케이스 |
|---|---|---|
| 전북은행 | 여신 RM | 대출 상담, 심사 준비, 사후관리 |
| 전북은행 | 전세 담당 | 전세보호, 전세사기 위험 검토 |
| 전북은행 | 이상거래 담당 | 보이스피싱, 이상거래 탐지 |
| JB우리캐피탈 | 여신 RM | 캐피탈 대출 상담, 심사 지원 |

이 구조를 통해 같은 플랫폼 안에서 여러 계열사와 여러 역할을 확장할 수 있다.

디자인 관점에서 중요한 원칙은 다음과 같다.

- 최상위 네비게이션은 도메인이 아니라 **계열사/역할**이다.
- 메인 콘텐츠는 역할별 **케이스 큐**다.
- 상세 화면은 **AI 추천 + 근거 + 승인 액션**을 중심으로 구성한다.
- 보조 패널은 **감사 로그, 개인정보, 리스크 근거**를 보여준다.

## 4. 주요 기능 정의(Key Features)

### 4.1 역할 기반 콘솔

- 사용자는 도메인이 아니라 자신의 역할로 진입한다.
- 운영자는 역할별 처리 현황, 승인 대기, 위험 케이스를 통합 확인한다.
- `계열사 x 역할` 기준으로 화면과 권한을 구성한다.

### 4.2 케이스 큐

- 오늘 처리해야 할 케이스 목록을 제공한다.
- 위험도, 긴급도, 예상 소요 시간, AI 추천 여부를 표시한다.
- 담당자는 우선순위에 따라 케이스를 열고 처리한다.

### 4.3 AI 분석 및 추천

- 케이스별 핵심 정보를 요약한다.
- 위험 신호를 탐지한다.
- 다음 액션을 추천한다.
- 담당자가 바로 확인할 수 있는 판단 근거를 제공한다.

### 4.4 근거 기반 승인 플로우

- 담당자는 AI 추천에 대해 승인, 보류, 반려, 추가 검토 요청을 선택한다.
- 담당자 코멘트를 함께 기록한다.
- 최종 판단 이력을 저장해 사후 검토와 감사에 활용한다.

### 4.5 감사 및 보안 패널

- 어떤 데이터가 사용됐는지 표시한다.
- 개인정보 및 PII 접근 이력을 관리한다.
- AI 추천과 사람 승인 로그를 분리 저장한다.

## 5. 데이터 및 기술 활용(Data & Tech Plan)

기술 구조는 모든 역할에 공통 적용되는 뼈대를 먼저 만들고, 역할별 케이스만 확장하는 방식이다.

공통 파이프라인:

```text
Case -> Agent -> Skill -> Evidence -> 승인 -> Audit + PII
```

활용 데이터:

- 고객 상담 이력
- 계약, 대출, 거래 데이터
- 내부 업무 규정
- 심사 기준 문서
- 이상거래 패턴
- 전세보호 관련 체크리스트
- 담당자 승인 및 반려 이력

기술 구성:

- LLM 기반 업무 에이전트
- 역할별 Skill 모듈
- RAG 기반 근거 검색
- read-only DB/API 연동
- PII 마스킹 및 접근 통제
- 승인 이력 및 감사 로그 저장

초기 구현에서 중요한 점은 AI가 원본 시스템을 직접 변경하지 않는다는 것이다. 본선 시연 단계에서는 **read-only 기반 추천 및 판단 보조**로 시작하고, 사람이 승인한 결과만 기록한다.

## 6. 사용자 시나리오(User Scenario)

### 6.1 시나리오 A: 전북은행 여신 RM

1. RM이 아침에 콘솔에 접속한다.
2. 오늘 처리할 대출 상담 및 사후관리 케이스가 우선순위별로 정렬된다.
3. AI가 특정 고객 케이스를 `추가 검토 필요`로 표시한다.
4. RM은 케이스 상세 화면에서 AI 요약, 위험 근거, 관련 문서를 확인한다.
5. AI는 `추가 서류 요청` 또는 `심사팀 전달` 같은 다음 액션을 추천한다.
6. RM은 추천을 승인하거나 수정한다.
7. 모든 판단 과정은 감사 로그로 남는다.

### 6.2 시나리오 B: 이상거래 담당자

1. 보이스피싱 의심 케이스가 위험도 높음으로 표시된다.
2. AI가 거래 패턴, 과거 유사 사례, 경고 근거를 요약한다.
3. 담당자는 차단, 보류, 고객 확인 요청 중 하나를 선택한다.
4. 최종 조치와 판단 근거가 기록된다.

### 6.3 시나리오 C: 운영자 어드민

1. 운영자는 전체 계열사와 역할별 케이스 처리 현황을 확인한다.
2. 승인 대기, 지연 케이스, 고위험 케이스를 한 화면에서 본다.
3. 특정 역할의 AI 추천 정확도와 담당자 승인 패턴을 비교한다.
4. 문제가 있는 케이스는 감사 로그와 근거 패널을 열어 검토한다.

## 7. 기대 효과 및 향후 확장성(Expected Impact)

기대 효과는 세 가지다.

1. **정확성 향상**
   - 흩어진 근거를 AI가 모아 담당자의 판단 품질을 높인다.

2. **업무 속도 향상**
   - 반복적인 문서 확인, 요약, 체크리스트 검토 시간을 줄인다.

3. **감사 가능성 확보**
   - AI 추천, 담당자 판단, 최종 승인 이력을 분리해 추적할 수 있다.

확장성 측면에서는 하나의 도메인 앱이 아니라 **역할 기반 콘솔 구조**이기 때문에, 전북은행에서 시작해 JB우리캐피탈, 그룹 내 다른 계열사로 확장할 수 있다.

초기에는 여신 RM, 전세 담당, 이상거래 담당 3개 역할로 시작하고, 이후 채권관리, 고객센터, 내부통제, 리스크관리 등으로 확장 가능하다.

## 8. 본선 구현 결과 및 시연 계획

본선에서는 완성형 플랫폼 전체가 아니라, **역할 기반 콘솔의 핵심 흐름이 실제로 작동하는 프로토타입**을 시연한다.

시연 범위:

1. 운영자 콘솔
   - 계열사/역할별 업무 현황
   - 승인 대기 케이스
   - 위험도 높은 케이스

2. 역할별 워크스페이스
   - 여신 RM 화면
   - 전세 담당 화면
   - 이상거래 담당 화면

3. 케이스 상세
   - AI 요약
   - 근거 문서
   - 추천 액션
   - 승인/보류/반려 버튼

4. 감사 로그
   - AI 추천 이력
   - 담당자 승인 이력
   - 사용 데이터 및 PII 처리 내역

발표에서 전달할 핵심 메시지:

> 우리는 도메인별 AI 앱을 여러 개 만든 것이 아니라, 금융 조직의 역할별 업무 판단을 하나의 콘솔 안에서 AI와 함께 처리하고 감독할 수 있는 구조를 만들었습니다.

## 9. 디자이너를 위한 화면 구성 메모

이 제품은 금융 업무 담당자를 위한 작업 도구이므로, 마케팅형 화면보다 **조용하고 조밀한 운영 콘솔**에 가깝게 설계한다.

권장 화면 구조:

- 좌측: 계열사/역할 네비게이션
- 중앙: 역할별 케이스 큐
- 우측: AI 요약, 근거, 리스크, 감사 로그 패널
- 상단: 검색, 필터, 승인 대기, 고위험 케이스 상태
- 상세: 케이스 정보, AI 추천, 근거 문서, 담당자 액션

디자인 원칙:

- 역할과 도메인을 같은 레벨에 놓지 않는다.
- 사용자가 먼저 이해해야 하는 것은 `내 역할에서 오늘 처리할 케이스`다.
- AI 추천은 단독 결론처럼 보이지 않게 하고, 항상 근거와 함께 보여준다.
- 승인/보류/반려는 화면의 가장 명확한 액션으로 배치한다.
- 감사 로그와 PII 처리는 보조 패널로 제공하되, 숨겨진 기능처럼 느껴지지 않게 한다.
