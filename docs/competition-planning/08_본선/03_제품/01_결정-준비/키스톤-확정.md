# 키스톤 — 콘솔 조직 축 (2026-07-03 종합)

> 근거: [[이승보-구현-설계-정리]](원문)·[[키스톤-역할축-검증]](리서치 D31·D3d·app.js·canon)·[[결정-현황-종합]]. 결정 자체는 팀 확정 대기 — 여기는 **뾰족한 권고 + 근거**.

## 🔑 키스톤 (한 문장)
> **콘솔의 1차 축 = [계열사(은행/비은행) × 담당 직군]. 도메인(여신·전세·피싱·사후관리…)은 그 직군이 처리하는 "케이스".**
> 이걸 잠그면 도메인 조합(Q1)·RM 포괄범위(Q2)·에이전트 구성(Q3)·데모 3케이스·실동작 구현 범위·은행/비은행 콘솔이 **연쇄로 정의**된다.

이미 팀이 이 방향으로 구현 중임(리프레임일 뿐, 새 방향 아님): 이승보님이 **본선용으로 역할별 대시보드를 신규 구현**(예선엔 단일 대시보드) — 왼쪽 **사이드바에서 담당자 클릭 → 그 직군 대시보드로 전환**. 계열사도 **은행(전북)+캐피탈 2트랙**으로 본인이 통합 제안.

## ⚠️ 리서치 교정 — "역할"은 실제 직군으로
초기 A안은 "전세 담당·이상거래 담당"을 별도 역할로 뒀으나 **1차 자료에 그런 직군은 없다**. 실제로 정렬:
- **실측 직군(D31)** = RM · 여신심사 · **사후관리** · 준법 · AML(이상거래) — 5개.
- **구현된 승인자(app.js `approvalLevelMatrix`·canon §2)** = **RM + 준법 2역할**.
- → **전세·피싱은 별도 직군이 아니라, 기존 직군이 다루는 도메인**(전세=여신심사/RM, 피싱=준법/AML). 사이드바 직군 = RM·여신심사·사후관리·준법/AML, 승인 게이트는 RM·준법으로 라우팅.

## 🏦 계열사 현실 (도메인은 계열사마다 다르다 = 모듈식)
- **전북은행(은행권)**: 여신(SOHO 대출심사, 히어로 전주카페)·전세보호·보이스피싱·사후관리.
- **JB우리캐피탈(비은행권, D3d)**: 오토·개인·부동산·기업·투자 여신 심사·**사후관리/부실징후 조기경보(EWS)**. ⚠️ **전세 상품 없음** — 전세는 은행 전용.
- → "3도메인이 두 계열사 모두로 확장"은 **전세에서 성립 안 함**. 진짜 확장 서사 = **"동일 운영계약(Case→…→Audit)·PII 4중방어 위에, 계열사별로 직군·도메인이 다르게 구성된다 = 모듈식 확장성."** (이게 "같은 3개 반복"보다 강한 5.3 확장성 논거.)

## 🧩 놓친 고가치 도메인 (리서치 발굴)
- **사후관리(자금용도외 유용 점검)** — **2026-06-30 시행 규제로 업무량 폭증 중**, JB 네이버클라우드 MOU가 명시한 "사후관리" 직결, 대환대출 1위 실적과 연결, **양 계열사 공통**. → **킬러 도메인, 편입 강력 권고.**
- **정책자금 매칭** — 이미 `Policy Match Agent` 존재, canon 대환대출 1위와 직결. 보조.
- **부실징후 조기경보(EWS)** — 우리캐피탈 연체율 2.21%·NPL과 직결(비은행 케이스로 적합).

## 도메인 조합 — 3옵션 (팀 택1)
| 옵션 | 3(±) 도메인 | 강점 | 약점 |
|---|---|---|---|
| **1. 계열사 대칭** | 여신 · 사후관리 · 이상거래(피싱) | 양 계열사 미러 → 확장 서사 깔끔, 전부 실제 직군 | 전세 감성 페인 상실 |
| **2. 페인 강도** | 여신 · 전세보호 · 피싱 | 공개 페인 강렬(전세 39,121건·피싱 1조) | 전세=은행전용(캐피탈 미러X), 전세·피싱 별도직군 아님 |
| **3. 하이브리드 【권고】** | 은행=여신(RM)·전세보호·피싱 / 캐피탈=여신·사후관리(EWS) | 계열사별 특화(현실 정합)+모듈식 확장 서사+감성페인 유지+사후관리 규제 훅 | 두 콘솔 도메인이 달라 "설명"이 한 줄 더 필요 |

→ **권고: 옵션 3(하이브리드) + 사후관리 편입.** 데모 3케이스(모두 실 LLM 에이전트 목표):
1. **전북은행 · RM/여신심사 · 여신** — 전주 카페 대출심사(히어로 JBG-104 risk88). *가장 깊게.*
2. **전북은행 · 준법/AML · 보이스피싱**(1조1,330억) *또는* **전세보호**(39,121건) — 감성·긴급 컷.
3. **JB우리캐피탈 · 사후관리 · EWS/자금용도 점검** — 비은행 확장 + 규제 훅(2026-06-30).

## 데모 구성 질문 답 (도메인 vs 역할)
양자택일 아님 → **직군(사이드바)으로 콘솔을 조직하고, 각 직군의 대표 도메인 케이스로 시연.** 공통 뼈대(Case→Agent→Skill→Evidence→승인(RM·준법)→Audit + PII 4중방어)는 **한 번만** 만들고 직군별 **에이전트셋·화면·데이터만 스왑** → 모듈식이라 3케이스 실동작이 현실적(밤샘 개발 전제).

## 정직한 전제 (실동작 = 오늘 밤 목표)
로컬모델(Ollama)·Claude API는 7/3 기준 **미연결**(API 막 확보, 로컬은 제안 단계). "3 실동작"은 **현재 완성이 아니라 개발 목표** — 토큰·AI·밤샘 여력으로 실현 가능하나 문서·발표엔 [목표/조건부]로 정직하게. 최소 1개(히어로)는 반드시 실 LLM 동작, 나머지는 골든패스 실동작 지향.

## 팀 결정 필요 (사람만)
1. 도메인 조합 옵션 1/2/3 택 (권고 3).
2. 데모 케이스 ②를 전세 vs 피싱 중 택 (감성 서사).
3. 사후관리 편입 여부.
4. 승인 직군 2개(RM·준법) 유지 vs 확대.
