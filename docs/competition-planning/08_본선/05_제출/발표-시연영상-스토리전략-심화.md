## 0. TL;DR (30초)

- **발표와 시연영상은 다른 산출물**이다. 발표는 *설득*(5분/15분·라이브 리스크), 영상은 *완성도 증명*(가산점 5점·5분·100MB·"설명서 느낌"). 스토리를 **따로** 짠다.
- **발표 추천**: 예선 = **P1 데모-퍼스트** + 슬로건 훅, 결선 = **P2 공감 아크**. 발표자 = **김민주(스토리·공감·클로징) · 김주용(데모 해설·논리·확장성) · 이승보(라이브 조작·기술 신뢰)**.
- **시연영상 추천**: **V1 "제품 설명서" 백본** + 30초 인간 훅 + **로컬모델 오프라인 실동작 컷**(V4 모먼트) 이식.
- **도메인(전세 vs 보이스피싱)은 미정(다른 세션 진행)** — 이 전략은 **둘 중 무엇이 실동작이어도 성립**하도록 설계했다.
- **관통 원칙 3개**: ① MVP가 실제로 돌아가면 상위권 바닥이 깔린다(회의 판단). ② 로컬모델·PII는 *말이 아니라 보여준다*. ③ 우리는 "은행 전문가"인 척하지 않고 "**기회를 주면 빠르게 AX를 만드는 젊은 실행자**"로 포지셔닝한다.

## 2. 원본 회의록에서 건진 전략 신호 (정리본에 없던 것)

> 아래는 **FULL STT 직접 인용**(요지). 발표/영상 스토리의 재료다.

**(S-1) 발표자·공감 문제 — 김민주가 캐리 [7/2 01:02~01:04]**
- 김주용(경상도): *"저기 경상도 남자애가 발표하는데… 그분들 마음에 조금이라도 불신을 줄 수 있지 않을까. 저는 전북·전라도와 아무 관련이 없다. 공감대가 부족하다."*
- 결론: *"이 팀에서 먹힐 사람은 민주님."* 김민주: *"사투리 해야겠네."*
- → **정서·지역 공감 아크는 김민주에게 배치**. 단, 김주용 경고(01:06): *"보이기엔 연기일 수 있어도 결국 공감을 끌어내야."* → 연기가 아니라 **진짜 공감**이어야 함.

**(S-2) 김민주 = 실제 타깃 고객 [7/2 01:16]**
- *"저 대출할 때 5,000만 원 청년 전세 대출했는데 바로 들어왔어요."* → **본인이 청년 전세 당사자**. 전세 도메인이면 **1인칭 실화**로 쓸 수 있는 강력한 진짜 공감 자산.

**(S-3) 리빌 훅 — 팀원 이름이 케이스 데이터 [7/2 01:11, 01:03]**
- 김주용: *"딱 저희 팀원들 이름이 케이스에 있는 거예요. '전주 카페 · 김민주' 딱. 그러면 '재미있다' 하면서 보겠죠."* → 데모 첫 화면 케이스 목록에 **우리 페르소나(전주 카페 사장 등)** 노출 = 각인·몰입 장치.

**(S-4) "주접(열정적 오버설명)"이 먹힌다 [7/2 01:12~01:13]**
- 김민주: *"어플에 대해 주접을 좀 더 떨어도 좋을 것 같아요. 더 하면 왜 써야 하는지 이유가 계속 나와요."* → 데모 해설은 **담백하지 말고, 각 단계의 '이래서 좋다'를 적극적으로** 짚는다.

**(S-5) 우리는 은행을 모른다 — 그게 오히려 포인트 [7/2 52:00~54:00, 01:34]**
- 반복된 불안: 인터뷰·데이터 없이 실제 은행 업무를 모른다.
- 이승보 리프레임: *"이 대회는 다양한 사람이 어떤 아이디어로 어떻게 해결할지에 초점."*
- 김주용: *"'너네 모르는 거 당연해, 이해해. 그런 기회가 있을 때 어떻게 할 수 있냐'를 보고 점수 주는 것."*
- → **포지셔닝: 은행 전문가 코스프레 금지. "기회를 주면 이 속도로 AX를 만든다"는 실행력/잠재력**을 판다.

**(S-6) 채용 퍼널이다 [7/3 50:12~50:47]**
- *"인재를 찾기 위해 마련… 거르고 거르는 지능형 서비스실. 3등까지는 무조건 눈독."* → 심사는 **제품만이 아니라 '이 사람들'을 본다**. 팀 역량·태도가 점수. 7/2 01:30: *"팀 퀄리티를 보여드려야."*

**(S-7) MVP 실동작 = 상위권 바닥 [7/2 01:35]**
- 김주용: *"실제로 모델이 돌아가고 산출물이 신뢰할 만하면 3등 이상 가능성 있다."* 배점 전부 20점 균등(01:36) → **한 축도 못 버린다**.

**(S-8) 로컬모델은 말이 아니라 보여준다 [7/3 15:12~15:50]**
- 오프라인(인터넷 끊겨도) 로컬 구동 = 보안 증명. *"말로 하는 거랑 보여주는 거랑 다르다."* → **인터넷 차단 상태에서 로컬모델 응답**을 라이브/영상으로.

**(S-9) 시연 자체는 매우 짧다 → 더미 + 실동작 1개 [7/3 33:51~34:14]**

**(S-10) 5분 = 4비트 [7/3 28:59]**
- 김민주: *"왜 만들었냐 · 뭘 만들었냐 · 어떻게 만들었냐 · 어떻게 쓰냐, 이 네 개."* → **발표 골격의 기본 리듬**.

**(S-11) 영상은 "설명서 느낌" [7/3 45:25~45:34]**
- 이승보: 판매 광고보다 *"설명서 느낌"* 선호 + 올라마 실연결 넣으면 완성도 점수 큼. → **영상 톤 = 차분한 데모 내레이션**, 단 도입 훅은 예외로 인간적.

## 3. 발표(Presentation) 스토리 후보

> 5분은 잔혹하다. **라이브 풀데모는 고위험** → 데모는 *영상 클립 백본 + 라이브 "와우" 1모먼트*(로컬모델 응답 or 키보드 승인)로 de-risk 권장. 각 후보는 5분 예선 기준, 결선 15분은 확장.

### P1 — 데모-퍼스트 (⭐ 예선 추천)
- **골격**: 훅 30s(슬로건+팀명·케이스 리빌 S-3) → **데모 3:00**(전주 카페 골든패스: 케이스→AI 판단[로컬모델 화면]→키보드 승인→결과물→감사체인) → 차별화 1:00(계열사 확장성 + 사람 최종승인·PII 비반출) → 클로징 30s.
- **강점**: MVP 완성도(축④)·주제적합(①) 직결. S-7의 "바닥" 확보. 4비트(S-10)와 정합.
- **리스크**: 데모 실패. → **폴백 녹화본 필수**(§8).
- **적합**: 예선 생존(상위 5 진입).

### P2 — 공감 아크 (⭐ 결선 추천)
- **골격**: 페인 스토리 1:00(김민주 지역·청년 전세 **실화** S-2, 부모 보이스피싱) → 데모 2:30 → 신뢰·보안 1:00(로컬 오프라인 S-8) → 비전 30s.
- **강점**: 혁신성·공감(①⑤)·차별화. 채용 퍼널(S-6)에서 '사람'이 남는다.
- **리스크**: 데모 시간 압박. 정서 과잉 시 '연기'로 보일 위험(S-1 경고) → 진정성 유지.
- **적합**: 결선(우승 다툼)·차별화가 필요한 순간.

### P3 — "빠른 실행자" 프레이밍 (정직·잠재력)
- **골격**: 정직 선언 15s(*"우리는 은행 내부를 다 알지 못합니다. 하지만—"*) → 실행 증거(며칠 만에 하네스·로컬 실동작) → 데모 → *"기회를 주면 이 속도로 AX를 만듭니다."*
- **강점**: S-5·S-6 정면 활용. 심사 성격(잠재력·인재)과 정렬. 다른 팀과 톤이 다름.
- **리스크**: 자칫 자신감 부족으로 오독 → **당당한 프레이밍 필수**(약점 고백이 아니라 강점 선언).
- **활용**: 단독보다 **P1/P2에 1~2줄로 이식**(오프닝 또는 Q&A 태도)하는 편이 안전.

### P4 — 평가 5축 정조준
- **골격**: 각 비트를 축①~⑤에 1:1 매핑, 축마다 근거 한 컷.
- **강점**: 채점 누락 0. 산출물이 본체(§1 주석)라는 점과 정합.
- **리스크**: 감동 약함·기계적. → **P1/P2의 서사 위에 '체크리스트'로 겹쳐** 쓰는 용도.

### P5 — 한 메시지 관통
- **골격**: 슬로건 하나("손은 놓고, 눈만")로 전체를 관통, 반복 각인.
- **강점**: 기억·차별화. 5분 압축에 유리.
- **리스크**: 깊이 부족. → **P1/P2의 훅·클로징 장치로 사용**.

**조합 권고**: 후보는 배타적이지 않다. **예선 = P1(뼈대) + P5(훅) + P3(1줄 정직) + P4(축 커버 점검)**, **결선 = P2(뼈대) + P4(15분 축 커버) + P3(Q&A 태도)**.

## 4. 시연영상(Demo Video) 스토리 후보  *(가산 5점·5분·100MB·설명서 톤)*

> 발표와 **별개 산출물**. 필수 커버(서비스 소개·시나리오·주요 기능·AI 판단 결과·힌트 활용)를 **빠짐없이** 담아야 가산이 안전(차등 1~5 가능성, 7/3 46:55).

### V1 — "제품 설명서" (⭐ 백본 추천)
- **흐름**: 서비스 1컷 소개 → 로그인·역할 대시보드 → 케이스 인입 → **에이전트 판단(로컬모델 화면 표시)** → 키보드 승인(스페이스 2회) → 결과물·고객 발송 → 감사체인.
- **강점**: 이승보 선호(설명서 톤 S-11)·필수요소 1:1 커버·완성도 점수 최적. 실패 위험 낮음(녹화·편집).
- **리스크**: 밋밋할 수 있음 → 아래 훅·오프라인 컷 이식으로 보완.

### V2 — "담당자의 하루"
- **흐름**: 아침에 50건 쌓인 콘솔 → 에이전트 분류 → 사람 승인 → 퇴근. 하루 서사.
- **강점**: 몰입·업무 맥락 전달. **리스크**: 설명서 톤에서 벗어남·시간 소모.

### V3 — "고객 공감 → 담당자 해결" 이중 시점
- **흐름**: 김민주-as-고객(전주 카페/청년 전세) 불안 30s → 담당자 콘솔이 에이전트로 해결 → 고객 안도.
- **강점**: 정서·차별화(S-1·S-2). **리스크**: 필수요소 시간 압박·"설명서"에서 이탈.

### V4 — "보안 증명" 중심
- **흐름**: **인터넷 차단** → 로컬모델 응답 지속 → PII 화면 밖으로 안 나감 → 감사로그. 보안이 주인공.
- **강점**: 금융 신뢰의 핵(S-8)을 영상 증거로. **리스크**: 단일 주제라 서비스 전모 약함.

**조합 권고**: **V1 백본 + V3의 오프닝 30초(인간 훅) + V4의 오프라인 실동작 컷 1개**. 필수요소는 V1이 보장, 기억점은 훅·오프라인 컷이 만든다.
