# 회의록-2026-06-30-아이디어회의

> JB금융그룹 Fin:AI Challenge **본선 아이디어 회의**(6/30 밤, 온라인). 원문(대외비)=[[원문-2026-06-30-아이디어회의-JB발췌]].
> ⚠️ 본 정리본은 **JB 대회 한정**. 화자 실명은 STT 추정(역할 매핑이 확실한 경우만 표기).

## 참석 / 맥락
- 팀 **망상궤도**: 김주용(운영·리서치·하네스) · 이승보(개발 리드) · 김민주(디자인) · 참석자1(보조) + **외부 협력자**(Codex 계정주, 초기 1주 공동개발).
- 제품 방향·타겟·차별점을 **팀이 합의**하고, 협업 도구(Syncthing·Obsidian·Codex)와 일정을 정렬한 첫 본선 아이디어 회의.

## 핵심 요약 (3줄)
1. **타겟 = JB 2계열사**(은행 계열사 1 + 비은행 JB우리캐피탈 1)의 **RM 에이전트** — 여신·여신 사후관리·보이스피싱·전세사기.
2. **차별점 = 확장성**: 단일 프로젝트가 아니라 그룹 계열사 시스템·데이터로 **유기적 확장**되는 설계. 단계적 청사진(팀→부서→회사→그룹).
3. **시연 = 로컬 모델 실동작**(은행 보안), **조직도 시각화가 메인 UI**, JB 웹 디자인 시스템 차용.

## 공유·결정
1. **제품 타겟 구조**: JB금융그룹 → 은행 계열사(전북은행·광주은행 중 **1개**) + 비은행 계열사(**JB우리캐피탈**) **2 타겟**. 각 계열사 **RM(여신 담당자)** 역할을 에이전트로 — "자동화를 넘어 **의사결정까지**". 페인포인트: 보이스피싱·여신 사후관리·전세사기.
2. **차별점 = 확장성(extensibility)**: "단일 프로젝트 납품"이 아니라 그룹 계열사 시스템·데이터가 유기적으로 연결·융합되는, **확장성을 염두에 둔 시스템**. 다른 팀과의 결정적 차별점. → **단계적 확장 청사진**(MVP → 팀 단위 테스트·갭 보완 → 부서 → 회사 → 전 그룹, 단계별 투자 대비 효용 제시).
3. **시장·가치 관점**: 마켓 규모·니즈 기반 진입 서사. 가치 마인드셋 "5000만원짜리 결과물을 만든다(100부터라도 압도적으로)".
4. **디자인**: **JB 웹 디자인 시스템 차용**(외주 제작, 매우 정제), "많이 빼는 것이 좋은 디자인", **에이전트 조직도 시각화 = 메인 인터페이스**(AI가 못 만드는 인간 친화 디테일), 와이어프레임 재작성.
5. **기술**: **로컬 모델 중시(은행 보안)** — 외부 클로드/코덱스 연결하되 **로컬 모델 실동작 시연**으로 "안전" 체감. **14개 에이전트 → 리서치 기반 역할 재정합**. 기존 은행 시스템 연동(데이터 API/MCP)·DB 설계 범위·배포 vs 로컬은 **미정**. 미래 방향=은행 특화 알고리즘/AI.
6. **협업 도구 온보딩**: **Syncthing**(팀 기기 실시간 양방향 동기화, 문서 공유) · **Obsidian**(본선 폴더 **MOC 링크 네비게이션** 시연) · **Codex/Claude** 터미널 실행 · **프롬프트 로그·통계 하네스** 공유. **보고 규약**: 작업 단위 완료 시 일일이 보고 말고 **디스코드/카톡 간단히**.
7. **팀·상금**: 외부 협력자(Codex 계정주, 초기 공동개발)와 **상금 분배 합의 완료**(N분의1 기준, 사업 시 공동 의향 OK). 역할 분담: 개발(이승보)·리서치·운영·취합(김주용)·디자인 시스템·조직도(김민주)·보조(참석자1). *(상세 합의는 대외비 원문.)*
