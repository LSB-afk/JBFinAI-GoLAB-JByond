# 회의록-2026-07-04-스코프확정 & 최종 스프린트 (예정)

## ✅ 이미 확정됨 (7/2) — 다시 논의 X
- **제품**: 담당자별 AI 협업 콘솔. 로그인 → 본인 역할 대시보드만.
- **UX(차별화)**: 키보드-퍼스트. 스페이스/방향키 검토·수락, 오승인 방지 **스페이스 2회**. 메시지 "**일은 에이전트가, 사람은 눈만**."
- **에이전트**: 노하우 스킬 + MCP + 은행 DB + 국가 오픈 API + LLM. **최종 승인·책임 = 사람**, **고객 PII 직접 미취급**.
- **발표**: 발표자 = **김민주**(지역 공감), 히어로 = **전주 카페 대출 심사**(예선 계승).
- **디자인**: JB 디자인 시스템·UX 라이팅 1차 완료.

## 🗳️ 지금 결정할 것 (D1~D6)

### D1. 계열사 — 데모에 뭘 넣나
- 옵션: (a) 전북은행 1개 (b) 광주은행 1개 (c) 전북+광주 둘 다 (d) 은행1 + JB우리캐피탈
- 💡 제안: **전북(전주 카페 히어로) + JB우리캐피탈 = 2계열사**, 광주는 확장 사례로만 언급.

### D2. 데모 도메인 In/Out — 4종 중 실제 시연 (🎨 Figma node 4797:2)
> 완료된 워터폴 리서치(딥프롬프트 27종)를 각 도메인 기능으로 "녹임". 통계=_canon §10 검증치.

**도메인 × 워터폴 리서치 → 시스템**
- **① 여신(소상공인·SME 대출심사, 히어로 전주 카페) 준비도★★★** — 리서치 D1a·D2·D6·D7a/b·D16·D14 → 위험도 판단(riskScore)·차주정보 통합·권고액션 문서·대환/재기지원. 통계: 자영업자 대출 1,064.2조·취약 42.7만(13.7%)·전북 대환 1위.
- **② 여신 사후관리 준비도★★** — D2·D15·D11·D3(네이버클라우드 MOU) → 상시 모니터링·조기 위험감지·재무자료 요약. 근거: JB 네이버클라우드 MOU("추진 방향").
- **③ 보이스피싱(이상거래 탐지) 준비도★★★** — D1b·D10·D12·D15 → 송금 설문→위험점수→차단·이상거래 탐지·노인 조기경보. 통계: 피해 1.13조(+56%)·건당 5,248만·50대+53%.
- **④ 전세사기(전세 보호) 준비도★★** — D1b·D7a(국토부 등기·실거래가)·D5a → 전세가율 분석·권리관계/보증보험 체크·위험 경보. 통계: 39,121건·HUG 4.49조→1.24조(-72%)·평균 1.3억.
- **전 도메인 공통 워터폴**: D11 케이스 메모리·D21 인텔리전스(공공데이터·Customer360·시뮬)·D15 승인루프·감사·D18 신뢰보정UX·D20/_04 담당자 스킬설계·D5a·D10·D12·D17 규제·보안·클린룸(PII 비반출)·D9 아키텍처·D13 MVP지표.

**선택 후보 조합 (많이)**: A)여신 단독 B)여신+보이스피싱 ⭐제안 C)여신+전세사기(대출↔전세·통계강함·데이터부담) D)여신+여신사후(라이프사이클·MOU·시각약함) E)여신+보이스피싱+전세사기(3종·완결성리스크) F)4종 전부(실동작2+조직도전시2·확장성 극대).

> 🎨 다른 결정(D1·D3·D4·D5·D6) 확장 후보 메뉴 = Figma node 4798:2.

### D3. 에이전트 수 — 14개 중 데모 투입
- 💡 제안: **판단→행동→검증 코어 3~5개** 실동작, 14개는 조직도에 계층으로(확장성).

### D4. 로컬 모델 · 구동 방식
- 옵션: 배포형(웹서버) / 로컬형(우리 컴퓨터) / 하이브리드 · 모델(EXAONE·Qwen 등)
- 💡 제안: **하이브리드** — 로컬 모델 실동작 시연(보안 체감)은 우리 컴퓨터, 나머지 흐름은 브라우저로 완주 우선.

### D5. DB · 실연동 범위
- 옵션: 정적/localStorage / 서버 API+영속 DB / 하이브리드 · 실 공공API vs 목업
- 💡 제안: **최소 하이브리드** — 핵심 경로(감사체인)만 영속, **1도메인 실 공공 API**(국토부/보이스피싱) + 나머지 목업.

### D6. 슬로건 + 발표 역할
- 슬로건: "일은 에이전트가, 사람은 눈만" → **키워드 1개**로 (예: "눈으로 일한다" / "Eyes-only" / "스페이스로 끝")
- 💡 제안: 발표=김민주, 조작=이승보, 해설·Q&A=김주용. 스토리 = 전주 카페 → 담당자 → 심사위원 공감.

## 🧭 AI 판단·제안 (참고 — 결정 아님, Figma node 4796:2)
**렌즈**: ① D-2 시간 → 적게·깊게·확실히 ② 평가 결정타=MVP 실동작+신뢰성 → 넓이보다 완주 ③ 최대 약점=도메인 지식 부족 → 정확 재현 대신 "작동·신뢰·서사"에 베팅.
- **D1** 전북(히어로)+JB우리캐피탈, 광주=조직도 확장 노드 → 은행 추가비용 0으로 그룹 확장성 증명.
- **D2** 여신(전주카페)+보이스피싱 2개만 실동작(측정 쉬움·감정 임팩트·데이터 리스크 회피), 전세사기·여신사후=로드맵.
- **D3** 실동작 4~5개(판단·행동·검증), 조직도엔 14개 → 다 만들면 다 부실=완결성 자해.
- **D4** 하이브리드: 앱=브라우저, PII 스텝만 로컬 EXAONE 실동작 → 보안 서사+완주 안정 동시.
- **D5** 최소 하이브리드: 정적+감사체인 영속, 1도메인만 실 공공API(은행 내부 DB는 접근 불가→목업이 정직).
- **D6** 슬로건 "손은 놓고, 눈만" / 발표=김민주·조작=이승보·해설=김주용.
- **한 줄**: "적게, 확실히 작동하는 MVP + 확장성·보안 서사" = D-2 상위권 최선 베팅.
