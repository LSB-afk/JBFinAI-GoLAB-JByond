# 회의록-2026-07-03-스프린트회의

## 참석 / 맥락
- 팀 **망상궤도**: 김주용(리더·문서·로컬모델·API) · 김민주(디자인·UX·발표·브랜딩) · 이승보(개발·설계) · 최영욱(발표 서포트).
- 준비된 결정 게이트(D1~D6, [[회의록-2026-07-04-스코프확정]])를 **검토·확정**하고, **공식 [[본선_안내문]]을 실시간 발견**해 발표·제출 구조를 파악한 스프린트.

## 핵심 요약 (3줄)
1. **스코프 확정** — 계열사 = **전북은행 + JB우리캐피탈 2개** / 구동 = **하이브리드(로컬모델 실동작 시연, 이승보 PC)** / 도메인 = **RM(여신)·전세사기·보이스피싱 3핵심**(실동작 1개+더미, 최종 내일 오전).
2. **역할 확정** — 이승보=설계·개발, 김민주=시연·발표·브랜딩, 김주용=문서·API·시각화, 최영욱=발표 서포트. 발표자=김주용·김민주.
3. **안내문 발견** — 발표 **예선 5분+2분 / 결선 15분+5분(상위 5팀), 발표 스킬 평가 없음(내용 승부)**. 시연영상 가점 → 무조건 제출. ⚠️ 직장인 자격 리스크.

## 공유·결정
1. **확정 재확인**: 담당자별 AI 협업 콘솔(로그인→역할 대시보드) · 키보드 퍼스트 UX · 에이전트 무기(스킬·MCP·은행DB·국가 오픈API·LLM) · 최종승인=사람·PII 비반출 · JB 디자인·UX 라이팅 1차 완료.
2. **D1 계열사 = 전북은행 + JB우리캐피탈 2개(확정)**: 그룹·광주 등 콘솔 다 만들면 과다 → 금융그룹≈은행 구조 동일하니 **메인을 전북은행으로 통합 + 만들던 우리캐피탈 디벨롭**. 은행/캐피탈 콘솔 차이는 AI 위임 설계.
3. **D2 도메인 = RM(여신)·전세사기·보이스피싱 3핵심(방향)**: 5개(RM·기업여신·전세보호·보이스피싱·내부통제·준법)에서 좁힘. 시연 매우 짧음 → **여신 더미, 전세사기 or 보이스피싱 중 1개 실동작**. RM은 포괄적이라 여신으로 대체 검토. **최종 확정 = 내일(7/4) 오전** 김주용·이승보 정리.
4. **D3 에이전트 수**: 실동작 4~5개·조직도 14개, 세부 판단 어려워 **AI에 설계 위임**(역할·메모리·데이터 접근권한·도메인 RAG).
5. **D4 구동 = 하이브리드(확정)**: 클로드/코덱스 연결 + **로컬 모델(Ollama·젬마 등) 실동작 시연**. **최종 테스트 = 이승보 PC**. 보안 서사 = "오프라인 로컬 구동"을 말이 아니라 시연으로.
6. **D5 DB·연동**: 정적/서버/하이브리드는 김주용·이승보 논의. **외부 공공 API 1개 실연결**(전세 실거래가 등) 시도 + 나머지 목업, 핵심 3개로 절제.
7. **D6 슬로건·브랜딩**: 시스템 이름·슬로건("손은 놓고 눈만" 후보) = **김민주 담당**.
8. **역할 분담(확정)**: 이승보=설계·개발 / 김민주=시연 시나리오·발표 구성·브랜딩 / 김주용=문서·정의서·로컬모델(젬마) 시도·외부 API·발표 시각화(Figma/간트) / 최영욱=발표 서포트. 발표자=김주용·김민주(둘 다 대비).
9. **안내문 발견(중요)**:
   - 발표 = 1·2차 라운딩 **5분+2분 Q&A**(A/B 그룹) → 최종(상위 5팀) **15분+5분 Q&A**. "발표 여러 번" 인지.
   - **심사에 발표 스킬 항목 없음** → 내용·완성도 승부("잘 만든 놈이 이긴다"). 꾸밀 시간 없이 내용 충실.
   - 제출 = 발표자료 PDF·기능명세서 PDF(3p)·GitHub(퀄리티 중요). **시연영상 선택·가점 → 무조건 제출**(5분·100MB). 데모 URL 필요.
